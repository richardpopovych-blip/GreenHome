# Project Plan – GreenHome

## 1. Проєкт  
**Назва:** GreenHome — інтелектуальна система моніторингу енергоспоживання у домі  
**Розробник:** Попович Річард Степанович  
**Опис:** Система, що дозволяє в реальному часі збирати дані енергоспоживання, аналізувати їх та надавати рекомендації для оптимізації витрат.

## 2. Мета та задачі  
**Мета:** Забезпечити можливість контролю та зменшення енергоспоживання в домогосподарстві.  
**Задачі:**  
- Встановити апаратні датчики та підключити їх до контролера.  
- Розробити програмну частину: збір, зберігання, аналіз даних.  
- Створити інтерфейс користувача.  
- Провести тестування та запустити систему в експлуатацію.

## 3. Етапи та строки  
| Етап | Опис | Строк |
|------|------|--------|
| Підготовчий | Уточнення вимог, формування беклогу | 1 тиждень |
| Апаратна частина | Встановлення датчиків, підключення, тестування | 2–3 тижні |
| Програмна частина | Реалізація функціоналу: бекенд, база даних, UI | 3–4 тижні |
| Тестування | Перевірка роботи системи, корекція помилок | 1–2 тижні |
| Запуск | Передача в експлуатацію, документація | 1 тиждень |
| Підтримка | Моніторинг, оновлення, вдосконалення | постійно |

## 4. Ролі та відповідальність  
- **Розробник/Автор:** Попович Річард Степанович — загальна відповідальність за проєкт.  
- **Куратор/науковий керівник:** підтримка, перевірка відповідності ТЗ.  
- **Тестувальник:** перевірка функціоналу та якості.  
- **Користувач:** експлуатація системи, зворотній зв’язок.

## 5. Комунікація  
- Використовуємо GitHub (репозиторій, issues, Pull Requests).  
- Щотижневі короткі звіти (стан задач, ризики, блокування).  
- Документація зберігається в репозиторії (наприклад, у папці `/requirements` + README).

## 6. Контрольні точки  
- Завершено підготовчий етап (базовий беклог готовий).  
- Апаратна частина: датчики встановлені, дані читаються.  
- Програмна частина: функціонал реалізовано.  
- Тестування завершено без критичних багів.  
- Система запущена в експлуатацію.

## 7. Управління ризиками  
Див. файл «RiskMatrix». Ризики постійно переглядаються; план мінімізації виконано.

## 8. Критерії успіху  
- Коректне зчитування та аналіз даних.  
- Система стабільно працює.  
- Користувач задоволений.  
- Проєкт завершено в межах строків та бюджету (за можливості).

## 9. Прийом результату  
Система працює протягом тестового періоду (наприклад, 1 тиждень) без критичних збоїв. Код, документація та інструкції передано в репозиторій. Акт виконання підписано розробником та керівником.
